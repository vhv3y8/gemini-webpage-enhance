# Gemini Chat Downloader - Project Summary

구글 제미나이(Gemini) 웹페이지에서 활성화된 대화 내역을 감지하여 사용자가 지정한 다양한 포맷(Markdown, Plain Text 등)으로 완벽하게 다운로드해 주는 구글 크롬 확장 프로그램 프로젝트입니다. 

기획부터 네이티브 UI 이식 및 클린 아키텍처 리팩토링까지 반영된 세부 구현 명세와 핵심 설계 철학을 요약 정리합니다.

---

## 1. 프로젝트 목적 및 요구사항
- **목적**: 제미나이 웹 서비스의 UI 레이아웃 흔들림에도 유연하게 대처하며 대화 내용을 완벽한 무손실 마크다운 문서 및 텍스트로 추출하여 보관하는 도구 개발.
- **핵심 요구사항**:
  - **유연한 확장성**: 타 브라우저(Firefox, Safari 등) 포팅과 새로운 포맷(BBCode, HTML, Org-mode 등) 추가가 언제든지 가능한 구조 설계.
  - **클린 아키텍처**: 비즈니스 흐름과 구체적 기술 스펙의 계층을 격리하여 의존성 역전 원칙(DIP) 관철.
  - **강건한 스크래핑**: 수시로 변경되는 Angular 해시 클래스명에 휩쓸리지 않고, 커스텀 시맨틱 태그 구조를 역추적하는 파싱 엔진 설계.
  - **데이터 기반 제어**: 출력 포맷의 상세 규칙(헤더 양식, 메타데이터 표기법, 역할 접두사 등)을 코딩이 아닌 선언적 JSON 설정 파일로 정의 및 파싱.
  - **네이티브 UI 이식**: 플로팅 버튼 같은 외부 이물질 느낌의 버튼을 배제하고, 제미나이 기존 우측 상단 더보기 드롭다운 메뉴 안에 완벽한 디자인 합성으로 기능 주입.

## 2. 클린 아키텍처 (Ports & Adapters) 구조
도메인 핵심 로직과 기술적 세부사항을 엄격하게 단방향 의존성으로 격리했습니다.

```
src/
├── core/                       # [Domain Layer] 비즈니스 로직 및 코어 사양
│   ├── models/
│   │   └── chat.ts             # 데이터 구조체 (ChatSession, ChatMessage, ExportResult)
│   └── ports/
│       ├── scraper.ts          # [Port] 입력을 위한 스크래퍼 경계 인터페이스
│       └── formatter.ts        # [Port] 출력을 위한 포매터 경계 인터페이스
├── adapters/                   # [Technical Layer] 구체적인 입출력 구현부 (플러그인 방식)
│   ├── scrapers/
│   │   └── gemini-dom.ts       # [Adapter] 제미나이 DOM 노드를 크롤링하는 스크래퍼
│   └── formatters/
│       ├── configurable.ts     # [Adapter] JSON 파일에 반응하여 출력을 조립하는 포매터
│       └── configs/            # 선언적 포맷 사양 명세 데이터
│           ├── markdown.json   # 마크다운 포맷 규칙
│           └── plaintext.json  # 텍스트 포맷 규칙
├── ui/                         # [Presentation Layer] 브라우저 인터페이스
│   └── menu-injector.ts        # 제미나이 네이티브 드롭다운 동적 추적 및 버튼 렌더러
└── content.ts                  # [Composition Root] 서비스 조립 및 크롬 확장 기동
```

## 3. 실제 페이지 UI 분석 및 스크래핑 극복 명세

### A. 대화 목록 및 Q&A 구조 분석
- 제미나이의 채팅 목록 트리는 `<infinite-scroller data-test-id="chat-history-container">` 하위에 여러 개의 대화 묶음 단위(`.conversation-container`)가 수평 병렬로 나열됩니다.
- 각 Q&A 묶음은 내부적으로 사용자의 프롬프트 컴포넌트 `<user-query>`와 모델의 응답 컴포넌트 `<model-response>`로 나뉩니다.
- **파서 극복 방안**: 
  - 특정 해시 클래스(`_ngcontent-ng-c2553993477` 등)는 빌드 환경마다 동적으로 생성되므로, 컴포넌트 태그 자체인 `<user-query>`와 `<model-response>`를 기준으로 요소를 색출하여 부모-자식-형제 순회 전략으로 매칭합니다.

### B. 사용자 프롬프트 영역 노이즈 소거
- `<user-query>` 내부에는 스크린 리더(웹 접근성)를 위해 화면에는 보이지 않는 라벨인 `"You said"` 텍스트가 `.cdk-visually-hidden` 및 `.screen-reader-user-query-label` 클래스로 숨겨져 있습니다.
- **파서 극복 방안**:
  - 활성화된 사용자 화면을 해치지 않기 위해 사용자 질의 DOM 노드를 완전히 복제(`cloneNode(true)`)한 뒤, 복제본에서 숨김 라벨 및 각종 UI 제어 버튼 요소를 완전히 찾아 지운(`remove()`) 상태에서 안전하게 텍스트 본문만 복원해 냅니다.

### C. 모델 응답 영역 요소 재구성 및 코드 블록 격리
- 모델의 서식은 `<message-content>`의 하위 마크다운 렌더링 컨테이너에 의해 Paragraph, Header, List, Code Block 등으로 렌더링됩니다.
- **파서 극복 방안**:
  - **인라인 서식 복원**: 단순 `innerText` 추출 시 누락되는 볼드체(`<b>`), 이탤릭체(`<i>`), 인라인 코드(`<code>`), 링크(`<a>`) 요소를 하위 깊이까지 탐색하여 `**bold**`, `[text](url)` 등의 규칙에 맞게 완전하게 마크다운 인라인 서식으로 조립해 냅니다.
  - **목록 보존**: 순서 있는 리스트(`OL`)와 순서 없는 리스트(`UL`)의 아이템(`LI`) 요소들을 식별하여 마크다운에 알맞은 기호(`*`, `1.`)로 역추적 표기합니다.
  - **코드 블록 노이즈 방지**: 구글 제미나이는 코드 블록 상단에 복사 기능용 헤더바나 버튼을 오버레이합니다. 이를 거르기 위해 `<pre>` 노드를 찾아 정확하게 그 하위에 선언된 `<code>` 요소의 텍스트 본문만 분리 격리 추출합니다. 추가적으로 `class="lang-*"` 또는 `class="language-*"` 속성을 스캔하여 코드가 작성된 프로그래밍 언어를 감지해 냅니다.

## 4. 네이티브 UI 이식 디테일 및 고려한 부분
- **이질감 0%의 네이티브 버튼 설계 및 정밀한 타겟 매칭**:
  - 플로팅 버튼은 페이지의 중요한 영역(예: 스크롤 하단 바, 채팅 프롬프트 박스 등)을 가릴 우려가 있습니다.
  - 제미나이 상단의 대화 설정 버튼을 누르면 동적으로 삽입되는 `<gem-menu role="menu">` 드롭다운의 렌더링 시점을 `MutationObserver`로 포착합니다.
  - 이 드롭다운 메뉴 내부에 구글의 머티리얼 심볼(Material Symbols) 아이콘(`download` 및 `description`), 폰트 패밀리, 호버 시 페이드 인 배경 색상 트랜지션, 구분선 투명도를 완벽히 이식한 `<gem-menu-item>` 노드를 수동으로 빌드 및 조립하여 주입했습니다.
  - **오주입 차단을 위한 타겟 특정화**: `gem-menu` 엘리먼트가 모델 변경 드롭다운 등 다른 UI 요소에서도 범용으로 쓰이므로, 상단 점 3개 메뉴에만 내보내기 버튼이 정확하게 주입되도록 탑레벨 컴포넌트 `<conversation-actions-icon>` 내에서 현재 활성화된(`aria-expanded="true"`) 버튼의 `aria-controls` ID를 동적으로 추출하고, 이 ID가 생성된 `gem-menu` 요소의 ID 혹은 관계 구조와 정확히 매칭할 때만 주입을 허용하는 이중 검증 구조를 설계하여 UI 주입의 강건성을 완벽히 확보했습니다.
- **다중 다운로드 양식 지원**:
  - 리팩토링된 JSON 설정 덕분에 마크다운 다운로드(**Export Markdown**)뿐만 아니라 텍스트 전용 내보내기(**Export Plain Text**) 항목까지 총 2개의 버튼을 구분선과 함께 이식하여 기능적 완성도를 극대화했습니다.
- **성능과 크기 최적화**:
  - 핵심 소스코드는 불필요한 서드파티 라이브러리를 단 하나도 도입하지 않고 오직 순수 바닐라 DOM API와 Web Utility를 기반으로만 구현하여, 컴파일 결과물 파일 크기를 **16.39 kB** 수준으로 초경량화하고 배포 안전성을 높였습니다.

## 5. 자동화 빌드 및 크로스 플랫폼 패키징 (`archiver` 통합)
- **Vite 커스텀 플러그인 구현**:
  - `vite.config.ts` 파일 내부에 `chromeExtensionZip` 커스텀 플러그인을 구현했습니다.
  - 빌드 프로세스의 마지막 수명 주기인 `closeBundle` 훅에 결합하여 작동합니다.
  - 외부 `archiver` 모듈의 기본 CJS 모듈 호환성과 TypeScript typings를 위해 패키지를 `archiver@7.0.1`로 고정하고, `import archiver from 'archiver'` 정적 임포트 규칙을 온전히 유지함으로써 IDE 경고가 없는 완벽한 타입을 획득했습니다.
- **압축 구조 및 최적화**:
  - `pnpm build:zip` 실행 시 `dist/` 내부의 빌드된 결과물(`content.js` 및 `manifest.json`)만 격리하여 ZIP 파일의 **루트 경로**에 삽입합니다. (크롬 웹스토어 업로드 규격 완벽 대응)
  - Zlib 레벨 9 최고 압축 효율을 적용하여 최종 패키지 아카이브 크기를 **4.79 KB** 수준으로 초경량 패키징하는 데 성공했습니다.

## 6. GitHub Actions 자동화 배포 파이프라인
- **지속적 제공 (CD)**:
  - `.github/workflows/release.yml` 파일을 통하여 완전 자동화된 빌드 및 릴리즈 배포망을 구축했습니다.
  - 개발자가 새로운 버전을 확정하고 태그(예: `v1.0.0`)를 푸시하면 액션이 자동으로 트리거됩니다.
  - GitHub 가상 빌드 환경에서 `pnpm` 의존성 캐싱을 탑재하여 15초 내외의 매우 신속한 속도로 확장 프로그램 빌드(`pnpm build:zip`)가 진행됩니다.
- **릴리즈 배포 자동화**:
  - 빌드가 끝나면 `softprops/action-gh-release`를 통해 GitHub Release에 정식 배포본을 자동 발행합니다.
  - 압축 완료된 `gemini-chat-downloader.zip` 파일이 자동으로 첨부되며, 커밋 내역으로부터 영리하게 변경사항(Changelog)을 추출하여 릴리즈 노트를 마크다운으로 자동 생성해 줍니다.
